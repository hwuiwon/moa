# Step L03 — Eval harness wiring + Grafana dashboards + regression alerts

_Bridge `moa-eval` to the lineage hypertable so retrieval-quality and grounding regressions surface continuously. Add Grafana dashboard JSONs, Prometheus alerting rules, and a `Score` record shape inspired by Langfuse's data model. Can run in parallel with L02 — touches separate files._

## 1 What this step is about

L01 and L02 capture lineage. L03 makes it useful for **operators** and the **eval harness**:

- Operators get Grafana panels showing retrieval health (zero-recall rate, top-K stability, fusion contribution), grounding health (verified-citation rate, NLI distribution), generation health (token usage, cost, finish reasons), and lineage-pipeline self-health (drops, lag, S3 export status).
- The eval harness consumes a `ScoreRecord` shape — the same record type whether generated by online judges (NLI verifier output, BM25 sufficiency) or offline replay (FActScore, RAGAS, custom evaluators). Scores are stored in the same hypertable, joined to turns by `turn_id`. Regression alerts fire when score distributions drift > 2σ over a 24h window.
- A `moa eval replay` command takes a saved dataset of turns and re-runs them through the current retriever/model/verifier stack, recording scores into `analytics.scores` for comparison.

L03 is **operationally lightweight** — it's bookkeeping on top of L01/L02, not new infrastructure.

## 3 Goal

After L03:

- `ScoreRecord` type is finalized in `moa-lineage-core` (replaces L01's placeholder Eval body).
- `analytics.scores` table exists alongside `analytics.turn_lineage`, joined via `(turn_id, dataset_run_id)`.
- `crates/moa-eval/` integrates with `LineageSink` — every eval run emits `ScoreRecord`s into the hypertable.
- Online-judge scores from L02's cascade verifier (per-citation `verified` flag, NLI entailment distribution) are also persisted as `ScoreRecord`s.
- `moa eval replay <dataset> --against=<model_config>` re-runs a saved dataset and records scores.
- `dashboards/grafana/` directory contains importable JSON dashboards: `moa-retrieval.json`, `moa-grounding.json`, `moa-generation.json`, `moa-lineage-self-health.json`.
- `ops/prometheus/alerts/` contains alert rules for: zero-recall spike, grounding-rate drop, lineage-drop spike, cold-tier export lag, citation hallucination spike.
- `cargo build --workspace` clean.

## 4 Rules

- **`ScoreRecord` is the single shape**, regardless of source. Online judges, offline replay, human review — all emit `ScoreRecord`s into the same table. This is non-negotiable; fragmenting score shapes makes dashboards a nightmare.
- **No new database** — `analytics.scores` is a sibling table in the same TimescaleDB. Continuous aggregates feed dashboards directly.
- **Dashboards are version-controlled JSON**, not configured manually in Grafana. Use `grafana-cli` import or kubectl `ConfigMap` for deployment.
- **Alert thresholds are tunable per workspace**, not globally. Different workspaces have different baselines.
- **Score writes go through the same `LineageSink`** as everything else. Don't add a parallel write path. The sink is async and at-least-once; that's fine for scores.
- **No external eval-platform dependency** (no LangSmith, no Phoenix Cloud). Self-hosted Grafana + Prometheus + the existing Postgres.

### 5a Finalize `ScoreRecord` in `moa-lineage-core`

Replace the L01 placeholder. In `crates/moa-lineage/core/src/records.rs`:

```rust
#[derive(Clone, Debug, Serialize, Deserialize)]
pub struct ScoreRecord {
    pub score_id: Uuid,
    pub ts: DateTime<Utc>,
    /// One of these MUST be set; mutually exclusive.
    pub target: ScoreTarget,
    pub workspace_id: WorkspaceId,
    pub user_id: Option<UserId>,
    pub name: String,                  // "grounding", "retrieval_recall", "citation_verified", "faithfulness", ...
    pub value: ScoreValue,
    pub source: ScoreSource,           // who/what produced this score
    pub model_or_evaluator: String,    // "hhem-2.1-open" | "gpt-4o-judge" | "human-reviewer-7" | ...
    pub run_id: Option<Uuid>,          // for offline replay groupings
    pub dataset_id: Option<Uuid>,      // for offline replay against a saved dataset
    pub comment: Option<String>,
}

#[derive(Clone, Debug, Serialize, Deserialize)]
#[serde(tag = "kind")]
pub enum ScoreTarget {
    Turn { turn_id: TurnId },
    Session { session_id: SessionId },
    DatasetRunItem { run_id: Uuid, item_id: Uuid },
}

#[derive(Clone, Debug, Serialize, Deserialize)]
#[serde(tag = "type", content = "v")]
pub enum ScoreValue {
    Numeric(f64),                      // e.g. NLI entailment 0.0..1.0
    Boolean(bool),                     // e.g. verified true/false
    Categorical(String),               // e.g. "supported" | "neutral" | "contradicted"
}

#[derive(Clone, Debug, Serialize, Deserialize)]
pub enum ScoreSource {
    OnlineJudge,                       // emitted live during a turn (e.g. cascade verifier)
    OfflineReplay,                     // emitted by `moa eval replay`
    Human,                             // emitted by manual review tooling
    External,                          // emitted by integrations (e.g. user thumbs-up/down)
}
```

Update `LineageEvent::Eval(serde_json::Value)` → `LineageEvent::Eval(ScoreRecord)`.

### 5b Extend the schema

Add to `crates/moa-lineage/sink/sql/schema.sql`:

```sql
CREATE TABLE IF NOT EXISTS analytics.scores (
    score_id          UUID         NOT NULL,
    ts                TIMESTAMPTZ  NOT NULL,
    workspace_id      UUID         NOT NULL,
    user_id           UUID,
    target_kind       TEXT         NOT NULL,    -- 'turn' | 'session' | 'dataset_run_item'
    turn_id           UUID,
    session_id        UUID,
    run_id            UUID,
    item_id           UUID,
    dataset_id        UUID,
    name              TEXT         NOT NULL,
    value_type        TEXT         NOT NULL,    -- 'numeric' | 'boolean' | 'categorical'
    value_numeric     DOUBLE PRECISION,
    value_boolean     BOOLEAN,
    value_categorical TEXT,
    source            TEXT         NOT NULL,    -- 'online_judge' | 'offline_replay' | 'human' | 'external'
    model_or_evaluator TEXT        NOT NULL,
    comment           TEXT,
    PRIMARY KEY (score_id, ts)
);

SELECT create_hypertable('analytics.scores', 'ts',
    chunk_time_interval => INTERVAL '1 day', if_not_exists => TRUE);

ALTER TABLE analytics.scores SET (
    timescaledb.compress,
    timescaledb.compress_segmentby = 'workspace_id, name',
    timescaledb.compress_orderby   = 'ts DESC'
);
SELECT add_compression_policy('analytics.scores', INTERVAL '7 days', if_not_exists => TRUE);
SELECT add_retention_policy   ('analytics.scores', INTERVAL '90 days', if_not_exists => TRUE);

CREATE INDEX IF NOT EXISTS ix_scores_workspace_name_ts
    ON analytics.scores (workspace_id, name, ts DESC);
CREATE INDEX IF NOT EXISTS ix_scores_turn ON analytics.scores (turn_id) WHERE turn_id IS NOT NULL;
CREATE INDEX IF NOT EXISTS ix_scores_run ON analytics.scores (run_id) WHERE run_id IS NOT NULL;

-- Continuous aggregate: hourly grounding rate per workspace
CREATE MATERIALIZED VIEW IF NOT EXISTS analytics.grounding_hourly
WITH (timescaledb.continuous) AS
SELECT time_bucket('1 hour', ts) AS bucket,
       workspace_id,
       AVG(CASE WHEN value_boolean THEN 1.0 ELSE 0.0 END) AS verified_rate,
       COUNT(*) AS n
FROM analytics.scores
WHERE name = 'citation_verified' AND value_type = 'boolean'
GROUP BY bucket, workspace_id
WITH NO DATA;
SELECT add_continuous_aggregate_policy('analytics.grounding_hourly',
    start_offset => INTERVAL '7 days',
    end_offset   => INTERVAL '5 minutes',
    schedule_interval => INTERVAL '5 minutes',
    if_not_exists => TRUE);

-- Continuous aggregate: hourly NLI entailment percentiles
CREATE MATERIALIZED VIEW IF NOT EXISTS analytics.nli_hourly
WITH (timescaledb.continuous) AS
SELECT time_bucket('1 hour', ts) AS bucket,
       workspace_id,
       APPROX_PERCENTILE(0.5,  PERCENTILEDISC(value_numeric)) AS p50,
       APPROX_PERCENTILE(0.95, PERCENTILEDISC(value_numeric)) AS p95,
       AVG(value_numeric) AS mean,
       COUNT(*) AS n
FROM analytics.scores
WHERE name = 'nli_entailment' AND value_type = 'numeric'
GROUP BY bucket, workspace_id
WITH NO DATA;
SELECT add_continuous_aggregate_policy('analytics.nli_hourly',
    start_offset => INTERVAL '7 days',
    end_offset   => INTERVAL '5 minutes',
    schedule_interval => INTERVAL '5 minutes',
    if_not_exists => TRUE);
```

The writer worker in `moa-lineage-sink` needs a second branch: when it sees `LineageEvent::Eval(ScoreRecord)`, route to `analytics.scores` with COPY. Same idempotency story — primary key `(score_id, ts)`.

### 5c Online judge integration

In L02's cascade verifier (`crates/moa-lineage/citation/src/cascade.rs`), each `VerifierResult` emit also fires a `ScoreRecord`:

```rust
fn emit_verifier_scores(
    sink: &dyn LineageSink,
    citation: &Citation,
    turn_id: TurnId,
    workspace_id: &WorkspaceId,
) {
    // citation_verified — boolean
    sink.record(LineageEvent::Eval(ScoreRecord {
        score_id: Uuid::now_v7(),
        ts: Utc::now(),
        target: ScoreTarget::Turn { turn_id },
        workspace_id: workspace_id.clone(),
        user_id: None,
        name: "citation_verified".into(),
        value: ScoreValue::Boolean(citation.verifier.verified),
        source: ScoreSource::OnlineJudge,
        model_or_evaluator: citation.verifier.method.clone(),
        run_id: None,
        dataset_id: None,
        comment: None,
    }));
    // nli_entailment — numeric (only if NLI ran)
    if let Some(ent) = citation.verifier.nli_entailment {
        sink.record(LineageEvent::Eval(ScoreRecord {
            score_id: Uuid::now_v7(),
            ts: Utc::now(),
            target: ScoreTarget::Turn { turn_id },
            workspace_id: workspace_id.clone(),
            user_id: None,
            name: "nli_entailment".into(),
            value: ScoreValue::Numeric(ent as f64),
            source: ScoreSource::OnlineJudge,
            model_or_evaluator: "hhem-2.1-open".into(),
            run_id: None,
            dataset_id: None,
            comment: None,
        }));
    }
}
```

Don't restructure the verifier — just call this helper inside its `verify_all` pass.

- `cost_micros`: numeric — emitted from each generation step.

### 5d Offline replay

`crates/moa-eval/src/replay.rs`:

```rust
pub struct ReplayConfig {
    pub dataset_id: Uuid,
    pub run_id: Uuid,
    pub model_override: Option<String>,
    pub embedder_override: Option<String>,
    pub limit: Option<usize>,
}

pub async fn replay_dataset(
    config: &MoaConfig,
    handles: SkillGraphHandles,    // see C04
    sink: Arc<dyn LineageSink>,
    cfg: ReplayConfig,
) -> Result<ReplayReport> {
    let items = load_dataset_items(&config.eval_pool().await?, cfg.dataset_id, cfg.limit).await?;
    let mut report = ReplayReport::new(cfg.run_id);

    for item in items {
        let turn_id = TurnId::new_v7();
        // 1. run retrieval against the current stack
        let retrieved = handles.retriever.retrieve_with_turn(&item.query, &item.scope, 10, turn_id).await?;
        // 2. run generation
        let answer = handles.provider.generate_with_turn(&item.query, &retrieved, turn_id).await?;
        // 3. score
        if let Some(expected) = &item.expected_answer {
            let f1 = bleu_or_f1(&answer, expected);
            sink.record(LineageEvent::Eval(ScoreRecord {
                score_id: Uuid::now_v7(),
                ts: Utc::now(),
                target: ScoreTarget::DatasetRunItem { run_id: cfg.run_id, item_id: item.id },
                workspace_id: item.workspace_id.clone(),
                user_id: None,
                name: "answer_f1".into(),
                value: ScoreValue::Numeric(f1),
                source: ScoreSource::OfflineReplay,
                model_or_evaluator: "f1-overlap".into(),
                run_id: Some(cfg.run_id),
                dataset_id: Some(cfg.dataset_id),
                comment: None,
            }));
        }
        report.items += 1;
    }
    Ok(report)
}
```

The dataset shape is minimal — `(item_id, workspace_id, scope, query, expected_answer?, expected_chunk_ids?)`. Store datasets in `analytics.eval_datasets` and `analytics.eval_dataset_items` (small, ordinary tables, not hypertables).

CLI subcommand: `moa eval replay --dataset=<uuid> --run-id=<auto>`.

### 5e Grafana dashboards

`dashboards/grafana/moa-retrieval.json`:

Panels:
- **Zero-recall rate** (per workspace): `analytics.turn_recall_hourly` continuous aggregate. Stat panel + time-series.
- **Top-K stability**: time-series of count distinct `payload #> '{retrieval,top_k,0}'` per hour per workspace (rough churn metric).
- **Vector hit count distribution**: histogram of `jsonb_array_length(payload #> '{retrieval,vector_hits}')`.
- **Rerank score distribution**: histogram of `payload #> '{retrieval,rerank_scores,0,relevance_score}'`.
- **Fusion contribution mix**: stacked area over time of `vector_contribution`, `graph_contribution`, `lexical_contribution` averages.
- **Retrieval latency p50/p95/p99**: from `payload #> '{retrieval,timings,total_ms}'`.
- **pgvector buffer hit ratio**: `buffers_hit / (buffers_hit + buffers_read)` averaged.
- **AGE edges-walked p95**: time-series.

`dashboards/grafana/moa-grounding.json`:

- **Verified citation rate**: from `analytics.grounding_hourly`.
- **NLI entailment p50/p95**: from `analytics.nli_hourly`.
- **Vendor-only vs verified breakdown**: pie chart by `model_or_evaluator`.
- **Citations-per-answer distribution**: histogram of `jsonb_array_length(payload #> '{citations}')`.
- **Citation positional bias**: heatmap of citation index → cited chunk position in `top_k`.

`dashboards/grafana/moa-generation.json`:

- **Tokens-per-turn p50/p95/p99**: input + output stacked.
- **Cost per turn**: average + 95p over time, segmented by provider.
- **Finish-reason breakdown**: stacked area (stop, length, content_filter, tool_calls, error).
- **Prefix-cache hit rate**: from `gen_ai.usage.cache_read.input_tokens / gen_ai.usage.input_tokens`.
- **Tool-call rate** by tool name.

`dashboards/grafana/moa-lineage-self-health.json`:

- **`moa_lineage_dropped_total` rate**: should be 0 in steady state; spike indicates undersized channel.
- **`moa_lineage_recorded_total` rate**: sanity check that capture is alive.
- **fjall journal depth** (custom metric to add): pending events not yet COPYed.
- **COPY batch size p95 / batch-flush latency p95**.
- **Cold-tier export lag**: `now() - max(last_ts) FROM analytics.lineage_export_progress`.
- **S3 PUT errors rate**.

Each panel uses the workspace_id template variable so operators can scope a dashboard to one tenant.

### 5f Prometheus alerting rules

`ops/prometheus/alerts/moa-lineage.yaml`:

```yaml
groups:
- name: moa-lineage
  interval: 60s
  rules:
  - alert: MOALineageDropping
    expr: rate(moa_lineage_dropped_total[5m]) > 0
    for: 5m
    annotations:
      summary: "Lineage events being dropped (channel saturated)"
      description: "{{ $value }} events/sec dropped on {{ $labels.instance }}. Increase channel_capacity or investigate writer backpressure."

  - alert: MOALineageWriterStuck
    expr: rate(moa_lineage_recorded_total[5m]) > 0 and rate(moa_lineage_flushed_total[5m]) == 0
    for: 10m
    annotations:
      summary: "Lineage writer not flushing"
      description: "Capture is alive but writer hasn't flushed in 10 minutes. Check Postgres connectivity / fjall disk space."

  - alert: MOAColdTierLagging
    expr: time() - moa_lineage_cold_last_export_timestamp_seconds > 7200
    for: 1h
    annotations:
      summary: "Cold-tier export >2h behind"
      description: "Last export at {{ $value }}; 30-day retention will drop data before archival. Investigate exporter."

- name: moa-grounding
  interval: 5m
  rules:
  - alert: MOAGroundingRateDropped
    expr: |
      (avg_over_time(moa_grounding_verified_rate[1h])
       < avg_over_time(moa_grounding_verified_rate[24h] offset 1d) - 2 * stddev_over_time(moa_grounding_verified_rate[24h] offset 1d))
       and avg_over_time(moa_grounding_verified_rate[1h]) < 0.7
    for: 30m
    annotations:
      summary: "Verified citation rate dropped >2σ for workspace {{ $labels.workspace_id }}"
      description: "Investigate retriever changes, embedder swap, or model change."

  - alert: MOAZeroRecallSpike
    expr: rate(moa_zero_recall_count[1h]) / rate(moa_turn_count[1h]) > 0.05
    for: 30m
    annotations:
      summary: "Zero-recall retrievals >5% in workspace {{ $labels.workspace_id }}"
      description: "Retriever returning empty top-K too often. Check vector index, embedder API, or sidecar health."

- name: moa-generation
  interval: 5m
  rules:
  - alert: MOACostPerTurnSpike
    expr: |
      avg_over_time(moa_cost_micros_per_turn[1h])
      > 2 * avg_over_time(moa_cost_micros_per_turn[24h] offset 1d)
    for: 30m
    annotations:
      summary: "Cost per turn doubled vs yesterday's baseline ({{ $labels.workspace_id }})"
```

Surfacing into Prometheus: add `metrics-exporter-prometheus` recorders for the rates referenced (`moa_grounding_verified_rate`, `moa_zero_recall_count`, `moa_turn_count`, `moa_cost_micros_per_turn`, `moa_lineage_flushed_total`, `moa_lineage_cold_last_export_timestamp_seconds`). Each is a thin reader that runs on a 60s tick, querying the continuous aggregates and exposing the result as a gauge.
