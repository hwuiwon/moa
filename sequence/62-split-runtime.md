# Step 62 — Split `moa-runtime` into Modules + `enum_dispatch`

_Break the 52KB single-file crate into focused modules. Eliminate ~600 lines of `match self { Local(r) => ..., Daemon(r) => ... }` boilerplate with the `enum_dispatch` crate._

---

## 1. What this step is about

`moa-runtime/src/lib.rs` is 52KB — the entire crate is one file. It contains three structs (`LocalChatRuntime`, `DaemonChatRuntime`, `ChatRuntime` enum), ~30 methods duplicated across the enum arms, plus daemon IPC helpers, relay functions, and utility functions. The `ChatRuntime` enum has the same mechanical delegation problem as `SessionDatabase` in `moa-session`: every method is `match self { Local(r) => r.foo(), Daemon(r) => r.foo() }`.

---

## 2. Files to read

- **`moa-runtime/src/lib.rs`** — The entire 52KB file. Understand the three structs, the ChatRuntime enum dispatch, and the free functions.
- **`moa-runtime/Cargo.toml`** — Current dependencies. Add `enum_dispatch`.
- **`src-tauri/src/commands.rs`** — Uses `ChatRuntime` extensively. Verify nothing breaks.
- **`moa-cli/src/main.rs`** — Uses `ChatRuntime`. Verify nothing breaks.

---

## 3. Goal

After this step:
1. `moa-runtime/src/lib.rs` is a slim re-export hub (~50 lines)
2. `LocalChatRuntime` lives in `local.rs` (~300 lines)
3. `DaemonChatRuntime` lives in `daemon.rs` (~350 lines)
4. Daemon IPC plumbing lives in `daemon_ipc.rs` (~200 lines)
5. Shared helpers live in `helpers.rs` (~100 lines)
6. `ChatRuntime` enum dispatch is generated by `enum_dispatch`, eliminating ~600 lines of hand-written `match` arms
7. Public API unchanged — all callers continue compiling without modification

---

## 4. Rules

- **Public API does not change.** `use moa_runtime::ChatRuntime` and all its methods must keep working.
- **Use `enum_dispatch` crate** to eliminate the mechanical `match self { Local(r) => ..., Daemon(r) => ... }` delegation. This means defining a trait with all the shared methods and having `enum_dispatch` generate the enum impl.
- **Do NOT change method signatures.** Move them verbatim.
- **Platform-gated (`#[cfg(unix)]` / `#[cfg(not(unix))]`) functions** move to `daemon_ipc.rs` together.

---

## 5. Tasks

### 5a. Add `enum_dispatch` to `moa-runtime/Cargo.toml`

```toml
[dependencies]
enum_dispatch = "0.3"
```

### 5b. Define the `ChatRuntimeOps` trait

Extract the shared method signatures from `ChatRuntime` into a trait. This is the interface that both `LocalChatRuntime` and `DaemonChatRuntime` implement:

```rust
use enum_dispatch::enum_dispatch;

#[enum_dispatch]
#[async_trait]
pub trait ChatRuntimeOps {
    fn session_id(&self) -> &SessionId;
    fn model(&self) -> &str;
    fn workspace_id(&self) -> &WorkspaceId;
    fn sandbox_root(&self) -> PathBuf;
    fn config(&self) -> &MoaConfig;
    async fn create_session(&self) -> Result<SessionId>;
    async fn set_workspace(&mut self, workspace_id: WorkspaceId) -> Result<SessionId>;
    async fn reset_session(&mut self) -> Result<SessionId>;
    async fn set_model(&mut self, model: String) -> Result<SessionId>;
    async fn session_meta(&self) -> Result<SessionMeta>;
    async fn session_meta_by_id(&self, session_id: SessionId) -> Result<SessionMeta>;
    async fn session_events(&self, session_id: SessionId) -> Result<Vec<EventRecord>>;
    async fn list_sessions(&self) -> Result<Vec<SessionSummary>>;
    async fn list_session_previews(&self) -> Result<Vec<SessionPreview>>;
    async fn list_memory_records(&self, filter: Option<MemoryRecordFilter>) -> Result<Vec<MemoryRecordSummary>>;
    async fn recent_memory_entries(&self, limit: usize) -> Result<Vec<MemoryRecordSummary>>;
    async fn search_memory(&self, query: &str, limit: usize) -> Result<Vec<MemorySearchResult>>;
    async fn read_memory_record(&self, id: &str) -> Result<MemoryRecord>;
    async fn write_memory_record(&self, record: MemoryRecordWrite) -> Result<MemoryRecord>;
    async fn delete_memory_record(&self, id: &str) -> Result<()>;
    async fn observe_session(&self, session_id: SessionId, event_tx: mpsc::UnboundedSender<SessionRuntimeEvent>) -> Result<()>;
    async fn queue_message(&self, session_id: SessionId, prompt: String) -> Result<()>;
    async fn soft_cancel_session(&self, session_id: SessionId) -> Result<()>;
    async fn hard_cancel_session(&self, session_id: SessionId) -> Result<()>;
    async fn respond_to_session_approval(&self, session_id: SessionId, request_id: Uuid, decision: ApprovalDecision) -> Result<()>;
    async fn run_turn(&self, prompt: String, event_tx: mpsc::UnboundedSender<RuntimeEvent>) -> Result<()>;
    async fn respond_to_approval(&self, request_id: Uuid, decision: ApprovalDecision) -> Result<()>;
    async fn cancel_active_generation(&self) -> Result<()>;
}
```

Note: `tool_names` and `tool_names_async` have different signatures between Local and Daemon — handle those as inherent methods on `ChatRuntime` rather than trait methods. Same for the static constructor methods (`from_config`, `attach_to_*`, etc.) — those stay on `ChatRuntime` directly since they construct the enum.

### 5c. Apply `#[enum_dispatch(ChatRuntimeOps)]` to `ChatRuntime`

```rust
#[enum_dispatch(ChatRuntimeOps)]
#[derive(Clone)]
pub enum ChatRuntime {
    Local(LocalChatRuntime),
    Daemon(DaemonChatRuntime),
}
```

This generates all ~25 `match self { Local(r) => ..., Daemon(r) => ... }` delegations automatically.

### 5d. Create module files

Move `LocalChatRuntime` and all its methods to `local.rs`.
Move `DaemonChatRuntime` and all its methods to `daemon.rs`.
Move all `#[cfg(unix)]` / `#[cfg(not(unix))]` daemon IPC functions to `daemon_ipc.rs`:
- `daemon_request`, `daemon_expect_ack`, `daemon_is_available`, `daemon_connect`, `daemon_send_command`
- `relay_daemon_runtime_events`, `relay_daemon_runtime_turn_events`

Move shared helpers to `helpers.rs`:
- `expand_local_path`, `local_user_id`, `last_session_message`, `start_empty_session`
- `relay_runtime_events`, `relay_session_runtime_events`, `unexpected_daemon_reply`

### 5e. Slim down `lib.rs`

```rust
mod daemon;
mod daemon_ipc;
mod helpers;
mod local;

pub use daemon::DaemonChatRuntime;
pub use helpers::{SessionPreview, SessionRuntimeEvent};
pub use local::LocalChatRuntime;

use enum_dispatch::enum_dispatch;
// ... ChatRuntimeOps trait definition
// ... ChatRuntime enum with #[enum_dispatch]
// ... Static constructor methods (from_config, attach_to_*, for_test)
// ... tool_names / tool_names_async inherent methods
```
