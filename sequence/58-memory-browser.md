# Step 58 — Memory Browser

_Two-pane wiki browser: tree view on the left, rendered markdown page on the right. Search, inline editing, page creation and deletion._

---

## 1. What this step is about

Build the memory browser view using prompt-kit's Markdown component for rendering and Source component for cross-references.

## 3. Goal

After this step:
1. Two-pane flex layout: tree (`w-[250px] shrink-0`) + content (`flex-1`)
2. Tree organized by page type folders: topics/, entities/, decisions/, skills/, sources/
3. Page renders using prompt-kit `<Markdown>` with frontmatter metadata header
4. Related pages displayed using prompt-kit `<Source>` component
5. Search, edit mode, create new page, delete page
6. Wiki links `[[page-name]]` are clickable

- **Use prompt-kit Markdown** for page content.
- **Use prompt-kit Source** for related pages.

### 5g. Create `use-memory-pages.ts` hooks

```typescript
import type { PageSummaryDto, WikiPageDto, MemorySearchResultDto } from '@/lib/bindings';

export function useMemoryPages(filter?: string) {
  return useQuery({
    queryKey: ['memory-pages', filter],
    queryFn: () => invoke<PageSummaryDto[]>('list_memory_pages', { filter }),
  });
}

export function useMemoryPage(path: string | null) {
  return useQuery({
    queryKey: ['memory-page', path],
    queryFn: () => invoke<WikiPageDto>('read_memory_page', { path }),
    enabled: !!path,
  });
}

export function useMemorySearch(query: string) {
  return useQuery({
    queryKey: ['memory-search', query],
    queryFn: () => invoke<MemorySearchResultDto[]>('search_memory', { query, limit: 20 }),
    enabled: query.length >= 2,
  });
}
```

Note: All DTO types come from `@/lib/bindings`, generated by ts-rs. If a field mismatch occurs, fix it in the Rust DTO and re-run `cargo test -p moa-app`.
